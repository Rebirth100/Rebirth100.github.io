rebirth1002026-07-182026-07-18

树与二叉树

一、树的基本概念

树（Tree） 是 n（n≥0）个结点的有限集合 T，它满足两个条件：

  • 有且仅有一个特定的称为根（Root） 的结点；
  • 其余的结点可以分为 m（m≥0）个互不相交的有限集合 T₁、T₂、……、Tₘ，其中每一个集合又是一棵树，并称为其根的子树（Subtree）。

树的表示方法主要有：树形表示法和目录表示法。

1. 基本术语

术语 说明
结点的度数 一个结点的子树的个数
树的度数 树中结点的最大度数
叶子结点（终端结点） 度数为零的结点
分支结点 度数不为零的结点
内部结点 除根结点外的分支结点
子结点 一个结点的直接后继结点
父结点（双亲结点） 直接后继结点的前驱结点
兄弟结点 同一父结点的各个子结点之间互为兄弟

2. 路径与层次

  • 路径：结点系列 k₁, k₂, …, kⱼ，满足 kᵢ 是 kᵢ₊₁ 的父结点。路径的长度为 j-1，即路径中的边数。
  • 祖先与子孙：路径中前面的结点是后面结点的祖先，后面结点是前面结点的子孙。
  • 结点的层数：根结点的层数定义为 1，其余结点的层数等于父结点的层数加 1。
  • 树的高度（深度）：树中结点层数的最大值。

3. 有序树与森林

  • 有序树：树中每个结点的各个子树的排列从左到右不能交换，即兄弟之间有序。一般的树是有序树。
  • 森林：m（m≥0）棵互不相交的树的集合。树去掉根结点就成为森林，森林加上一个新的根结点就成为树。

二、二叉树的概念

二叉树（Binary Tree） 是 n（n≥0）个结点的有限集合，它或者是空集（n=0），或者由一个根结点以及两棵互不相交的、分别称为左子树和右子树的二叉树组成。

二叉树与普通有序树的关键区别：二叉树严格区分左孩子和右孩子，即使只有一个子结点也要区分左右。

三、二叉树的特性

  1. 满二叉树：深度为 k（k≥1）的二叉树有 2ᵏ-1 个结点。
  2. 完全二叉树：只有最下面两层有度数小于等于 2 的结点，且最下面一层的叶结点集中在最左边的若干位置上。
  3. 具有 n 个结点的完全二叉树的深度为：⌊log₂n⌋ + 1。
  4. 深度为 k（k≥1）的二叉树最多有 2ᵏ-1 个结点。

  5. 重要性质：在任意一棵二叉树中，叶子结点的数目比度数为 2 的结点的数目多一。

    推导过程：

    • 总结点数为各类结点之和：n = n₀ + n₁ + n₂
    • 总结点数也为所有子结点数加一：n = n₁ + 2·n₂ + 1
    • 两式联立得：n₀ = n₂ + 1

四、完全二叉树的顺序存储

1. 编号方法

完全二叉树结点的编号方法是从上到下、从左到右，根结点为 0 号结点。

2. 顺序存储特点

假设完全二叉树的结点数为 len，某结点编号为 i：

关系 计算公式
父结点编号（i > 0） (i-1)/2
最后叶子结点的父结点索引 len/2 - 1
左子结点索引 2i + 1
左兄弟（i 为偶数且 i ≠ 0） i - 1
右兄弟（i 为奇数且 i < len-1） i + 1

注意：不完全二叉树通过添加虚结点构成完全二叉树，再用数组存储，会浪费一些存储空间。

六、二叉树的遍历

遍历：对树中的每一个结点访问一次且仅访问一次。

由于二叉树的递归性质，遍历算法通常也是递归的。

1. 三种遍历方式

遍历方式 访问顺序
先序遍历（Preorder） 先访问树根 → 再访问左子树 → 最后访问右子树
中序遍历（Inorder） 先访问左子树 → 再访问树根 → 最后访问右子树
后序遍历（Postorder） 先访问左子树 → 再访问右子树 → 最后访问树根

八、哈夫曼树及应用

1. 基本概念

术语 说明
路径 从一个结点到另一个结点之间的分支序列
路径长度 从一个结点到另一个结点所经过的分支数目
结点的权 赋予树中每个结点的具有某种实际意义的实数
带权路径长度 从树根到某结点的路径长度与该结点的权的乘积
树的带权路径长度（WPL） 树中所有叶子结点的带权路径长度之和

2. 哈夫曼树（最优二叉树）

由 n 个带权叶子结点构成的所有二叉树中，带权路径长度 WPL 最短的二叉树称为哈夫曼树。

构造步骤：

  1. 初始化：将每个字符及其权值看作一棵单结点的树，放入最小优先队列（最小堆）。
  2. 循环合并：当队列中多于一棵树时：
    • 从队列中取出权值最小的两棵树；
    • 创建新结点，权值为两树权值之和；
    • 将取出的两棵树作为新结点的左右子树（左子树权值 ≤ 右子树）；
    • 将新树放回队列。
  3. 结束：队列中只剩一棵树时，即为哈夫曼树。

3. 哈夫曼编码

哈夫曼编码是一种最优前缀编码（任意一个编码不能成为其他任意编码的前缀），用于无损数据压缩。

生成步骤：

  • 在哈夫曼树基础上，约定左分支 = 0，右分支 = 1（也可相反）；
  • 从根到每个叶子结点的路径上的二进制串，即为该字符的哈夫曼编码。

示例（字符频率：A:5, B:9, C:12, D:13, E:16, F:45）：

字符 频率 路径（从根出发） 哈夫曼编码
F 45 左 0
C 12 右→左→左 100
D 13 右→左→右 101
A 5 右→右→左→左 1100
B 9 右→右→左→右 1101
E 16 右→右→右 111

编码示例：字符串 “FACE” → F(0) + A(1100) + C(100) + E(111) = 01100100111

解码：从二进制串逐位读取，遇 0 走左、遇 1 走右，到达叶子结点时输出对应字符，然后回到根继续。

九、扩展：AVL树、红黑树、B树、B+树

文档中提及了 AVL树、红黑树、B树、B+树，这些是更高级的树结构，分别应用于不同的场景：

  • AVL树：自平衡二叉搜索树，通过旋转保持平衡。
  • 红黑树：一种近似平衡的二叉搜索树，广泛应用于 C++ STL map/set、Java TreeMap 等。
  • B树：多路平衡搜索树，常用于数据库和文件系统。
  • B+树：B树的变种，所有数据存放在叶子结点，内部结点仅作索引，广泛用于数据库索引。
