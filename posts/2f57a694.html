正则表达式零基础到精通（完整版无删减手册）.md

前言

本文为全网最完整零基础正则教程，从零讲解所有正则语法、规则、坑点、实战案例、企业常用模板、代码示例。全程可直接复制测试，无删减、无保留、无遗漏，学完可应对所有开发、爬虫、数据清洗、表单校验、面试场景。

正则全称：Regular Expression
适用语言：Python / JS / Java / Go / PHP / SQL 通用（细微差异文末标注）

推荐在线测试工具：https://regex101\.com/

一、正则核心本质

正则就是一套字符匹配规则，用来：

  • 判断字符串是否符合规则（手机号、邮箱、密码校验）

  • 批量提取文本内容（提取数字、URL、IP、中文）

  • 批量替换、清洗、过滤文本

  • 按复杂规则分割字符串

核心思想：用符号描述字符形态与位置

二、基础匹配（字面量匹配）

直接写普通字符，为精确匹配，区分大小写。

示例正则：hello

匹配：hello、123hello、xhello666
不匹配：Hello、hell、helloo

三、全部元字符（正则核心，必须熟记）

元字符是正则特殊符号，不能直接匹配自身，需要转义。

完整元字符列表：. * + ? ^ $ \ | ( ) [ ] { }

3.1 任意字符点 .

.匹配除换行符以外的任意单个字符

示例：a.b 匹配 aab、a1b、a@b、axb

3.2 转义符 \

将特殊元字符转为普通字符，需要匹配以下符号必须加 \：

\. \* \+ \? \^ \$ \\ \| \( \) \[ \] \{ \}

示例：a\.b 只匹配 a.b

3.3 起止位置符 ^ $

  • ^：字符串开头

  • $：字符串结尾

^abc：以abc开头
abc$：以abc结尾
^abc$：整串严格等于abc

四、字符集 [] 与范围匹配

4.1 基础字符集

[abc]：匹配 a / b / c 任意一个字符

4.2 范围简写

  • [0-9]：所有数字

  • [a-z]：所有小写字母

  • [A-Z]：所有大写字母

  • [a-zA-Z0-9]：字母+数字

  • [a-zA-Z0-9_]：单词字符

4.3 反向字符集 [^]

[^0-9]：匹配所有非数字字符
[^a-z]：匹配非小写字母

五、预定义字符类（高频简写）

全部通用简写，无需手写范围

符号 等价范围 含义
\d [0-9] 数字
\D [^0-9] 非数字
\w [a-zA-Z0-9_] 单词字符
\W [^a-zA-Z0-9_] 非单词字符
\s 空白符 空格、制表符、换行
\S 非空白符 可见字符

六、量词系统（控制匹配次数）

量词只修饰紧挨在前的一个字符或分组

6.1 基础量词

  • *：0次或多次（任意次数，可有可无）

  • +：1次或多次（至少1次）

  • ?：0次或1次（可选）

6.2 精准量词 {}

  • {n}：严格 n 次

  • {n,}：最少 n 次，无上限

  • {n,m}：n ~ m 次

示例：
\d{3} 三位数字
\d{6,} 至少6位数字
\d{3,8} 3~8位数字

七、贪婪匹配 \& 非贪婪匹配（重难点）

7.1 贪婪模式（默认）

尽可能匹配最长结果

文本：a123b456b
正则：a.*b
结果：a123b456b

7.2 非贪婪模式（加 ?）

量词后加 ?，匹配最短结果

正则：a.*?b
结果：a123b

常用万能非贪婪：.*?、.+?

八、分组、捕获与非捕获组

8.1 普通分组 ()

将多个字符视为整体，可配合量词

(ab)+ 匹配 ab、abab、ababab

8.2 捕获分组

分组内容会单独保存，可提取、复用

正则：(\d{4})-(\d{2})-(\d{2})
匹配 2025-12-31
组1：2025 组2：12 组3：31

8.3 非捕获分组 (?:)

只分组、不保存、性能更高，爬虫/批量匹配必用

(?:abc)+

九、或逻辑 |

| 代表或，优先级最低，建议括号包裹

cat|dog 匹配 cat 或 dog
(jpg|png|gif) 匹配三种图片后缀

十、单词边界 \b

\b 单词边界，只匹配位置，不占字符

用于精准整词匹配，防止部分匹配

\bkey\b 只匹配独立单词 key，不匹配 keys、akey

十一、零宽断言（高阶核心、面试必考）

断言只判断前后条件，不消耗字符，不占用匹配结果

11.1 正向先行断言 (?=)

匹配后面必须是指定内容的字符

\d+(?=元)：匹配“元”前面的所有数字

11.2 负向先行断言 (?!)

匹配后面不是指定内容的字符

\d+(?!元)：后面不带“元”的数字

11.3 正向后行断言 (?\<=)

匹配前面是指定内容的字符

(?<=价格)\d+：提取“价格”后面的数字

11.4 负向后行断言 (?\<!)

匹配前面不是指定内容的字符

十二、修饰符（全局通用标记）

  • i：忽略大小写

  • g：全局匹配（匹配所有结果）

  • m：多行模式，^$ 匹配每行首尾

  • s：让 . 匹配换行符

十三、企业生产级正则模板（全量可直接用）

13.1 国内手机号

^1[3-9]\d{9}$

13.2 通用邮箱

^\w+([-+.]\w+)*@\w+([-.]\w+)*\.\w+([-.]\w+)*$

13.3 18位身份证

^\d{17}[\dXx]$

13.4 合法IP地址

^(?:(?:25[0-5]|2[0-4]\d|[01]?\d\d?)\.){3}(?:25[0-5]|2[0-4]\d|[01]?\d\d?)$

13.5 HTTP/HTTPS URL

https?://[\w\-]+(\.[\w\-]+)+[\w\-\.,@?^=%&:/~+#]*

13.6 匹配纯中文

[\u4e00-\u9fa5]

13.7 强密码（6-16位，字母+数字）

^(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d]{6,16}$

13.8 匹配空白行

^\s*$

13.9 匹配手机号+固话

(0\d{2,3}-\d{7,8})|(1[3-9]\d{9})

十六、正则最全避坑指南（90%人踩的坑）

  1. 所有元字符必须转义：. * + ? ( ) [ ] { } \ | $ ^

  2. Python 必须使用 r"" 原始字符串，防止反斜杠转义冲突

  3. 默认贪婪匹配，爬虫取内容必须用 .*? 非贪婪

  4. [] 内部大部分元字符失效，无需转义

  5. ? 在量词后是非贪婪，在普通位置是0/1次，语义不同

  6. 校验完整字符串必须加 ^$，否则会匹配部分内容

  7. 零宽断言不占字符，适合只判断不截取的场景

十七、正则完整速查表（终极汇总）

字符类

\d \D \w \W \s \S .

量词

* + ? {n} {n,} {n,m} *? +? ??

位置

^ $ \b \B

分组与逻辑

() (?:) | []

断言

(?=) (?!) (?<=) (?<!)

修饰符

i g m s

十八、从零到精通学习路线（唯一正确顺序）

1. 字面量、元字符、字符集、预定义字符
2. 全部量词 + 贪婪/非贪婪
3. 分组、或逻辑、边界
4. 四大零宽断言（高阶）
5. 熟记所有行业通用正则模板
6. 熟练 Python/JS 正则 API
