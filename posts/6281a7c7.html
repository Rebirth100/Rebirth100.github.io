2.5 特殊数据类型

ENUM 枚举

  • 占用：1~2字节
  • 规则：1-255个选项占1字节；256-65535占2字节
  • 底层存储数字索引，而非字符串

SET 集合

  • 占用：1~8字节
  • 规则：
    1-8个成员：1字节 | 9-16：2字节 | 17-24：3字节 | 25-32：4字节 | 33-64：8字节

BIT 位类型 BIT(M)

占用计算公式：(M+7)/8 向上取整

  • BIT(1) → 1字节
  • BIT(8) → 1字节
  • BIT(9) → 2字节
  • BIT(64) → 8字节

JSON 类型

二进制存储，占用空间≈文本长度+元数据，适合存储复杂结构化数据

MySQL DDL与DML详解

一、DDL与DML核心区别速查表

特性 DDL (数据定义语言) DML (数据操作语言)
定义 定义数据库结构 操作数据库表数据
操作对象 数据库、表、索引、视图 表中的行记录数据
核心命令 CREATE, ALTER, DROP, TRUNCATE, RENAME SELECT, INSERT, UPDATE, DELETE
作用范围 修改库表结构 增删改查表内数据
事务特性 自动提交，无法回滚（绝大多数场景） 支持事务，可手动提交/回滚
锁粒度 通常表级锁 InnoDB下通常行级锁
使用频率 低频，多用于库表设计、维护 高频，业务日常操作
使用人群 DBA、后端架构师 开发人员、业务系统
通俗类比 建筑师，设计房屋框架 住户，使用房屋存放物品

二、DDL 数据定义语言

1. CREATE 创建数据库对象

用于新建数据库、数据表、索引、视图等结构

2. ALTER 修改已有对象结构

修改表字段、新增字段、删除字段、修改表名、调整字段属性

3. DROP 删除数据库对象

彻底删除结构+内部所有数据，不可恢复

4. TRUNCATE 清空表数据（DDL类型）

仅删除表内全部数据，保留表结构定义

5. RENAME 重命名数据表

三、DML 数据操作语言

仅操作表中记录，不修改表结构，业务最常用

1. SELECT 查询（核心DML）

2. INSERT 插入新增数据

3. UPDATE 更新已有数据

必须搭配WHERE条件，否则会全表更新

4. DELETE 删除表内记录

仅删除行数据，表结构保留

四、关键补充说明

  1. 权限限制：修改其他用户密码，账号至少需要 CREATE USER 权限，或 mysql.user 表的 UPDATE 权限、目标用户的 GRANT OPTION；
  2. 认证插件区别：mysql_native_password 兼容低版本客户端、老旧程序；caching_sha2_password 是新版默认，安全性更高，但部分老工具不支持；

    • 命令行尽量使用 -p 交互式输入密码，避免明文暴露；
    • 禁止直接 UPDATE mysql.user 明文写入密码；
  4. 密码生命周期：可通过 PASSWORD EXPIRE INTERVAL N DAY 强制用户定期更换密码，提升数据库安全。

创建students表，字段：studentno（主键）、name、sex、hometown、age、class、card（身份证，允许NULL）

2. 主键作用

  1. 唯一标识每行，保证数据不重复；

  6. 方便主从复制、日志追踪数据变更。

3. 常用条件查询语句

  1. 查询小乔年龄：字符串值必须用单引号，双引号默认识别为字段名

二、DDL 和 DML 核心区分

对比项 DDL（数据定义语言） DML（数据操作语言）
作用 定义/修改数据库、表、索引等结构 增删改查表内行数据
核心命令 CREATE、ALTER、DROP、TRUNCATE、RENAME SELECT、INSERT、UPDATE、DELETE
事务 自动提交，无法回滚 支持事务，可commit/rollback
锁 表级锁 行级锁(InnoDB)
使用场景 建库建表、改表结构、删库删表 业务日常数据读写

1. DDL 常用操作

  1. CREATE：创建数据库、表、索引、视图
  2. ALTER：新增/删除/修改字段、修改表名
  3. DROP：彻底删除数据库、表、索引（结构+数据全部清除）
  4. TRUNCATE：清空表所有数据，保留表结构（DDL，不可回滚）
  5. RENAME：修改表名称

2. DML 常用操作

  1. SELECT：单条件/多条件/模糊/排序/聚合统计查询
  2. INSERT：全字段插入、指定字段插入、批量插入
  3. UPDATE：按条件批量更新字段值
  4. DELETE：按条件删除指定行数据

MySQL 子查询补充 + 多表查询完整总结

一、SELECT 子查询 & 子查询分类

1. SELECT 列内子查询（每行单独计算）

直接写在SELECT后，每行都会执行一次子查询，用于附加统计字段：

  1. 计算员工工资与全公司平均薪资差值
  2. 统计每个部门下员工人数

2. 子查询按返回值分类

  1. 标量子查询：返回单个值（单行单列），用于数值比较
  2. 列子查询：返回一列多行，搭配IN / NOT IN使用
  3. 行子查询：返回一行多列，多字段匹配
  4. 表子查询：返回多行多列，可放在FROM作为派生表

二、多表查询基础

1. 演示三张关联表

  • departments：部门表（主键dept_id）
  • employees：员工表（dept_id外键关联部门，manager_id自关联员工）
  • projects：项目表（dept_id外键关联部门）
    三张表通过dept_id建立关联关系。

2. 6种连接方式详解

（1）INNER JOIN 内连接（可省略INNER）

只返回两张表关联字段匹配的数据，无匹配则不展示。
语法：表1 JOIN 表2 ON 关联条件

  • 单关联：员工+部门
  • 多表关联：员工 JOIN 部门 JOIN 项目
    区分：关联匹配条件写ON，数据过滤条件写WHERE

（2）LEFT JOIN 左连接

保留左表全部数据，右表无匹配字段填充NULL
常用场景：

  1. 查询所有员工（含无部门员工）
  2. 查询无部门的员工：WHERE 右表主键 IS NULL

（3）RIGHT JOIN 右连接

保留右表全部数据，左表无匹配字段填充NULL
常用场景：查询所有部门（含无员工部门），筛选无员工部门用WHERE 左表主键 IS NULL

（4）FULL JOIN 全连接

MySQL原生不支持，使用LEFT JOIN UNION RIGHT JOIN模拟，合并左右所有数据，去重。

（5）CROSS JOIN 交叉连接

生成两张表笛卡尔积（所有数据无序配对），无关联条件；等价于逗号分隔多表 FROM a,b。

（6）SELF JOIN 自连接

一张表自己和自己连接，必须设置两个不同别名：

  1. 查询员工与直属经理（manager_id关联自身emp_id）
  2. 查询同部门不同员工，用emp_id < e2.emp_id避免重复配对

三、核心语法规范

  1. 多表连接关联条件统一写在ON后，过滤条件放WHERE；
  2. 自连接必须定义两个表别名区分；
  3. LEFT/RIGHT JOIN 筛选“无匹配数据”时，判断右/左表主键IS NULL；
  4. UNION 合并查询结果会自动去重；
  5. INNER JOIN 只展示两边都存在的关联数据，丢失孤立数据。

    2 字段值为NULL时，正则匹配结果恒为false，不会被查询出来；
  2. 中文在[[:alpha:]]中会被识别为字母；
  3. 复杂匹配优先用分组()划分范围，避免|作用域混乱；
  4. 需要严格区分大小写时，通过COLLATE utf8mb4_bin指定二进制排序规则。
