链表（Linked List）

1. 概念与优势

1.1 数组 vs 链表

特性 数组 链表
随机访问 效率高（O(1)） 效率低（O(n)）
长度 固定，难以动态扩展 不固定，可动态增减
中间插入/删除 效率低（需移动元素） 效率高（O(1)）
空间 连续内存 不一定连续，分散存储

1.2 链表的本质

链表是一种链式存储结构，由一个个节点（Node）组成。每个节点通过 malloc 动态分配内存，存储相同类型的数据。

节点的本质是一个结构体，包含两部分： - 数据域（data field）：存储实际数据 - 指针域（pointer field）：存储指向下一个节点的指针

2. 链表分类

链表可按三个维度进行分类：

分类方式 类型
有无头结点 带头结点链表 / 无头结点链表
指针数量 单向链表 / 双向链表
是否循环 循环链表 / 非循环链表

组合举例：带头结点的单向非循环链表 是最常见的形式。

总结

操作 时间复杂度 关键点
遍历 O(n) 从头到尾逐个访问
按值查找 O(n) 遍历比较
头插 O(1) 直接修改头指针
尾插 O(n) 需遍历到末尾
中间插入 O(n) 查找位置 O(n) + 插入 O(1)
删除 O(n) 查找前驱 O(n) + 删除 O(1)

注意事项： 1. 插入/删除后记得更新指针链接，防止链表断裂 2. 删除节点后要 free 释放内存 3. 使用 -> 访问结构体指针成员 4. 头指针传参时注意二级指针的使用（需要修改头指针时）
