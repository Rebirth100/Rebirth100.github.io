Ⅱ、字符集字节占用对照表

同字符集单字符占用：

字符集 中文汉字 英文 / 数字 特殊说明
utf8mb4 (推荐) 3-4 字节 1 字节 MySQL现代默认，支持全部emoji/Unicode
utf8(Mysql内置)=utf8mb3 3 字节 1 字节 不支持4字节表情字符
gbk/gb2312 2 字节 1 字节 国内中文编码
latin1 不支持中文 1 字节 仅西欧字符
utf16 2-4 字节 2-4 字节 宽字符存储
utf32 固定4字节 固定4字节 所有字符统一4字节

字节基础规则

  1. 整数：1/2/4/8字节，INT最常用(4字节)
  2. 浮点数：FLOAT4字节、DOUBLE8字节
  3. DECIMAL精确小数：变长，占用高于普通整数
  4. utf8/utf8mb4：英文数字1字节，汉字3字节

Ⅲ、完整数据类型详解

1. 数值类型

1.1 整数类型

数据类型 存储空间 有符号范围 无符号范围
TINYINT 1 字节 -128 ~ 127 0 ~ 255
SMALLINT 2 字节 -32,768 ~ 32,767 0 ~ 65,535
MEDIUMINT 3 字节 -8,388,608 ~ 8,388,607 0 ~ 16,777,215
INT / INTEGER 4 字节 -2,147,483,648 ~ 2,147,483,647 0 ~ 4,294,967,295
BIGINT 8 字节 -9.22e18 ~ 9.22e18 0 ~ 18.45e18

1.2 浮点与定点小数

类型 存储空间 精度 说明
FLOAT 4字节 单精度7位有效数字 近似存储，不适合金额
DOUBLE 8字节 双精度15位有效数字 近似存储
DECIMAL(M,D) 变长 精确小数 M总位数，D小数位，财务专用

2. 日期时间类型

类型 存储空间 格式 取值范围
YEAR 1字节 YYYY 1901 ~ 2155, 0000
DATE 3字节 YYYY-MM-DD 1000-01-01 ~ 9999-12-31
TIME 3字节 HH:MM:SS[.微秒] -838:59:59 ~ 838:59:59
DATETIME 5~8字节 YYYY-MM-DD HH:MM:SS 1000-01-01 ~ 9999-12-31
TIMESTAMP 4字节 YYYY-MM-DD HH:MM:SS 1970-01-01 ~ 2038-01-19

3. 字符串类型

3.1 定长 CHAR(M)

  • 存储：M × 字符集最大字节，最大255字符
  • 特性：长度不足自动补空格，浪费空间
  • 示例 CHAR(10) utf8mb4：存1个英文字母也占用40字节

3.2 变长 VARCHAR(M)

存储公式：实际数据字节 + 长度前缀

  • 最大长度≤255字节：前缀1字节
  • 最大长度＞255字节：前缀2字节
  • 上限65535字节
    示例 VARCHAR(100) utf8mb4：
  • “hello”：5数据+1前缀=6字节
  • “你好”：6汉字字节+1前缀=7字节

3.3 大文本TEXT系列

类型 前缀字节 最大容量
TINYTEXT 1字节 255字节
TEXT 2字节 65535字节
MEDIUMTEXT 3字节 16,777,215字节
LONGTEXT 4字节 4GB级别

3.4 二进制BLOB系列

类型 存储规则 最大长度
BINARY(M) 定长M字节 255
VARBINARY(M) 实际长度+前缀 65535
TINYBLOB 数据+1字节前缀 255
BLOB 数据+2字节前缀 65535
MEDIUMBLOB 数据+3字节前缀 16777215
LONGBLOB 数据+4字节前缀 4GB

4. 特殊数据类型

4.1 ENUM 枚举（单选）

ENUM('small','medium','large')

  • 1~255个选项：占用1字节
  • 256~65535个选项：占用2字节
  • 底层存储数字索引，不存字符串

4.2 SET 集合（多选）

SET('a','b','c','d')

  • 1-8个成员：1字节
  • 9-16：2字节
  • 17-24：3字节
  • 25-32：4字节
  • 33-64：8字节

4.3 BIT 位类型 BIT(M)

占用计算：(M+7)/8 字节

  • BIT(1) → 1字节
  • BIT(8) → 1字节
  • BIT(9) → 2字节
  • BIT(64) → 8字节

4.4 JSON类型

二进制存储，占用≈文本长度+元数据，支持JSON内置查询函数

3. 主键作用说明

  1. 唯一标识每行记录，杜绝重复数据
  2. 自动生成索引，大幅提升主键查询速度
  3. 可作为外键关联其他数据表，维护参照完整性
  4. 约束：主键字段不允许NULL、值唯一
  5. InnoDB引擎以主键作为聚集索引，决定数据物理存储顺序
  6. 主从复制、日志同步依赖主键追踪数据变更
