MySQL 正则表达式完整学习笔记

十、MySQL中的正则表达式

1. 正则表达式操作符

1.1 REGEXP / RLIKE（两者等价）

用于判断字符串是否匹配正则表达式，部分匹配即可（只要字符串中包含匹配片段就返回 true）。

1.2 NOT REGEXP / NOT RLIKE

用于判断字符串不匹配正则表达式。

2. 基本元字符

元字符 描述 MySQL示例
. 匹配任意单个字符 'a.c' 匹配 “abc”、”aac”、”axc”
^ 字符串开始 '^abc' 匹配以 “abc” 开头的字符串
$ 字符串结束 'xyz$' 匹配以 “xyz” 结尾的字符串
* 0次或多次 'ab*c' 匹配 “ac”、”abc”、”abbc”
+ 1次或多次 'ab+c' 匹配 “abc”、”abbc” 但不匹配 “ac”
? 0次或1次 'ab?c' 匹配 “ac” 或 “abc”
{n} 恰好n次 'a{3}' 匹配 “aaa”
{n,} 至少n次 'a{2,}' 匹配 “aa”、”aaa” 等
{n,m} n到m次 'a{2,4}' 匹配 “aa”、”aaa”、”aaaa”

3. 字符类

3.1 自定义字符类

使用 [] 匹配括号中的任意一个字符，[^] 匹配不在括号中的任意一个字符。

注意：MySQL 正则是部分匹配，只要字符串中有一部分匹配，整个字符串就匹配。

3.2 预定义字符类

使用 [[:xxx:]] 格式调用预定义字符类：

大小写敏感：

  • MySQL 默认不区分大小写（排序规则如 utf8mb4_general_ci，ci = Case Insensitive）
  • 如需区分大小写，可使用 COLLATE utf8mb4_bin：
COLLATE 类型 含义 示例
utf8mb4_general_ci 不区分大小写 'A' = 'a'
utf8mb4_bin 区分大小写（二进制比较） 'A' ≠ 'a'
utf8mb4_unicode_ci 不区分大小写（更准确的 Unicode 规则） 'A' = 'a'

4. 分组和或运算

4.1 分组 ()

使用 () 对表达式分组，可用于重复或提取子模式。

4.2 或运算 |

使用 | 匹配多个模式。

5. 转义字符

特殊字符（如 .、\、() 等）需要转义才能匹配字面量，MySQL 中需用 \\ 转义（因为字符串本身会解析一次 \）。
